## Scope Statement (In-Scope)
The ISMS covers all systems, services, people, and processes involved in:

- Credit analysis and enforcement (Analyzer)
- Document and response storage (Vault)
- Entity and document mapping (threeb_id, context_type, context_id)
- Role-based access control and authorization
- Audit logging, monitoring, alerting, and compliance enforcement
- Client, operator, creditor, and reseller interactions
- Operational dashboards and scheduled jobs (KPI refresh, SLA monitoring)

## Included Systems
- Supabase PostgreSQL (RLS, encryption at rest, backups)
- Supabase Storage (private buckets: threeb-documents, threeb-responses)
- Supabase Edge Functions (vault-upload, vault-download, vault-complete, refresh-kpi)
- Application services (Analyzer + enforcement workflows)
- Observability systems (system_events, vault_events, Grafana dashboards, alerts)
- Portfolio scoping for reseller views (portfolio membership tables)

## Exclusions
None. All data processing, storage, and access are in-scope.

## Security Boundaries
- TLS for all data in transit
- Encryption at rest (platform-managed)
- Row Level Security (RLS) per threeb_id / portfolio scope
- Signed URLs for storage access (short-lived)
- Role- and mapping-based authorization
- Principle of least privilege and defense-in-depth
